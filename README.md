
# 🎮 gameserver-scripts

Scripts, recursos y plugins para servidores de juegos. Actualmente centrado en FiveM y Minecraft, con contenido propio que uso o he desarrollado para mis propios servidores.

---

## 📁 Contenido

```
gameserver-scripts/
├── fivem/        # Recursos y scripts para servidores FiveM
└── minecraft/    # Plugins y configuraciones para servidores Minecraft
```

---

## 🚔 FiveM

Los recursos de FiveM están desarrollados en Lua siguiendo la estructura estándar de recursos. Cada uno incluye un `fxmanifest.lua` y un README con instrucciones de instalación.

```
fivem/
└── nombre-recurso/
    ├── fxmanifest.lua
    ├── client.lua
    ├── server.lua
    └── README.md
```

---

## ⛏️ Minecraft

Plugins y configuraciones compatibles con Spigot/Paper. Cada carpeta indica la versión de Minecraft compatible.

```
minecraft/
└── nombre-plugin/
    ├── plugin.jar
    ├── config.yml
    └── README.md
```

---

## 🚀 Instalación

Cada recurso o plugin tiene su propio README con instrucciones detalladas. Lee siempre la documentación antes de instalar en producción.

# 🎮 gameserver-scripts

Scripts, resources and plugins for game servers. Currently focused on FiveM and Minecraft, with content I use or have developed for my own servers.

---

## 📁 Content

```
gameserver-scripts/
├── fivem/        # Resources and scripts for FiveM servers
└── minecraft/    # Plugins and configurations for Minecraft servers
```

---

## 🚔 FiveM

FiveM resources are developed in Lua following the standard resource structure. Each one includes a `fxmanifest.lua` and a README with installation instructions.

```
fivem/
└── resource-name/
    ├── fxmanifest.lua
    ├── client.lua
    ├── server.lua
    └── README.md
```

---

## ⛏️ Minecraft

Plugins and configurations compatible with Spigot/Paper. Each folder indicates the compatible Minecraft version.

```
minecraft/
└── plugin-name/
    ├── plugin.jar
    ├── config.yml
    └── README.md
```

---

## 🚀 Installation

Each resource or plugin has its own README with detailed instructions. Always read the documentation before installing in production.
